## 2. 两个核心数据结构

### 2.1 PlotSpec —— 所有线都要理解

PlotSpec 是一个**扁平 dict**，是三条线之间传递绘图意图的唯一载体。

**必填字段（缺少任何一个系统会回问用户，不会渲染）：**

| 字段 | 类型 | 合法值 | 示例 |
|------|------|--------|------|
| `chart_type` | str | `"bar"` `"line"` `"scatter"` `"box"` `"heatmap"` | `"bar"` |
| `data_source` | str | CSV文件路径 | `"data/example_bar.csv"` |
| `data_x` | **str** | **列名字符串**（不是列的值） | `"method"` |
| `data_y` | **str** | **列名字符串**（不是列的值） | `"accuracy"` |
| `style_theme` | str | `"nature"` `"ieee"` `"neurips"` `"clean"` `"morandi"` | `"nature"` |

> **⚠️ 高频错误：`data_x` / `data_y` 填的必须是列名，不是列的值。**
> 
> 错误示例：`"data_y": [93.5, 94.8, 91.0]`（这是列的值）  
> 正确示例：`"data_y": "accuracy"`（这是列名）

**可选字段（system/merger.py 会自动填入默认值，不填也不影响校验）：**

| 字段 | 默认值 | 含义 | 适用图表 |
|------|--------|------|----------|
| `data_group_by` | `null` | 分组列名，用于多系列 | bar, line |
| `data_error` | `null` | 误差棒列名 | bar, line |
| `data_filter` | `null` | 行过滤条件（pandas query 语法） | 全部 |
| `label_title` | `""` | 图表标题 | 全部 |
| `label_x` | `""` | X轴标签 | 全部 |
| `label_y` | `""` | Y轴标签 | 全部 |
| `axes_x_scale` | `"linear"` | X轴刻度类型 `"linear"` / `"log"` | 全部 |
| `axes_y_scale` | `"linear"` | Y轴刻度类型 | 全部 |
| `axes_y_min` | `null` | Y轴最小值 | 全部 |
| `axes_y_max` | `null` | Y轴最大值 | 全部 |
| `axes_x_tick_rotation` | `0` | X轴刻度旋转角度（度） | 全部 |
| `style_palette_override` | `null` | 配色覆盖 `"morandi"` `"nature_d"` `"tab10"` `"coolwarm"` | 全部 |
| `params_orientation` | `"vertical"` | 柱方向 `"vertical"` / `"horizontal"` | **bar** |
| `params_stacked` | `false` | 是否堆叠 | **bar** |
| `params_sort` | `null` | 排序 `"asc"` / `"desc"` / `null` | **bar** |
| `params_show_values` | `false` | 柱顶显示数值 | **bar** |
| `params_markers` | `true` | 显示数据点标记 | **line** |
| `params_smooth` | `false` | 曲线平滑 | **line** |
| `params_alpha` | `0.8` | 点透明度 | **scatter** |
| `params_show_regression` | `false` | 显示回归线 | **scatter** |
| `params_show_points` | `"outliers"` | 显示数据点策略 `"all"` / `"outliers"` / `"none"` | **box** |
| `params_notch` | `false` | 缺口箱线图 | **box** |
| `params_annot` | `true` | 单元格数值标注 | **heatmap** |
| `params_fmt` | `".2f"` | 数值格式字符串 | **heatmap** |

**合法 PlotSpec 完整示例：**

```json
{
  "chart_type": "bar",
  "data_source": "data/example_bar.csv",
  "data_x": "method",
  "data_y": "accuracy",
  "style_theme": "nature",
  "data_group_by": "dataset",
  "data_error": "std",
  "label_title": "模型准确率对比",
  "label_x": "模型",
  "label_y": "准确率 (%)",
  "axes_y_min": 50,
  "axes_y_max": 100,
  "axes_x_tick_rotation": 30,
  "params_sort": "desc",
  "params_show_values": true
}
```

### 2.2 DataContext —— A线需要，B线生成

DataContext 是 `tools/loader.py` 把 CSV 解析后生成的**可读摘要字符串**，用途是注入到模型 prompt。

**格式规范（固定，A线训练数据的 instruction 里必须用这个格式）：**

```
数据摘要：
- 形状：24行 × 5列
- 列信息：
  · method（类别型，唯一值6个）：BERT-base, RoBERTa-base, XLNet-base, ALBERT-base
  · dataset（类别型，唯一值4个）：SST-2, MR, CR, CoLA
  · accuracy（数值型，范围55.1~95.1）
  · std（数值型，范围0.3~1.1）
- 前2行预览：[["BERT-base","SST-2",93.5,0.3],["BERT-base","MR",87.3,0.4]]
- 缓存key：cache://a1b2c3d4
```

用 `data/example_bar.csv` 实际调用 `load_data()` 可以看到真实输出。

## 3. A线：模型推理

**负责文件：`model/generator.py`（唯一文件）**  
**不能动的文件：除 `model/generator.py` 之外的所有文件**

### 3.1 你要替换的东西

打开 `model/generator.py`，函数体现在是 Mock，你只替换函数体内部：

```python
def generate_spec(
    user_input: str,
    data_context: str,
    current_spec: dict | None = None,
) -> dict:
    # ← 替换这里，签名一行都不能改
```

**签名的含义：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `user_input` | str | 用户原始自然语言，如"画个柱状图对比各模型准确率" |
| `data_context` | str | DataContext 摘要字符串（格式见 [2.2节](#22-datacontext--a线需要b线生成)） |
| `current_spec` | dict 或 None | 首轮为 `None`；修改轮传入当前完整 PlotSpec |

**返回值约定：**

- `current_spec is None`（首轮）：返回包含所有 5 个必填字段的完整 PlotSpec dict
- `current_spec is not None`（修改轮）：只返回变更字段的 delta dict，例如仅 `{"style_theme": "ieee"}`
- 返回的必须是 **dict**，不能是 JSON 字符串

### 3.2 微调数据格式

每条训练样本结构：

```json
{
  "instruction": "<DataContext摘要>\n\n用户需求：<user_input>",
  "output": "<合法的PlotSpec JSON字符串>"
}
```

**首轮样本示例：**

```json
{
  "instruction": "数据摘要：\n- 形状：24行 × 4列\n- 列信息：\n  · method（类别型，唯一值6个）：BERT-base, ...\n  · accuracy（数值型，范围55.1~95.1）\n\n用户需求：画一张柱状图，比较不同模型的准确率，用nature风格",
  "output": "{\"chart_type\": \"bar\", \"data_source\": \"data/example_bar.csv\", \"data_x\": \"method\", \"data_y\": \"accuracy\", \"style_theme\": \"nature\"}"
}
```

**修改轮样本示例（output只包含变更字段）：**

```json
{
  "instruction": "...\n\n当前图表已显示。用户修改需求：换成ieee风格",
  "output": "{\"style_theme\": \"ieee\"}"
}
```

### 3.3 关键注意事项

1. `data_x` / `data_y` 强化训练：1.5B 模型的已知失败模式是把列的值填入这两个字段。专项样本里要大量展示"正确 = 列名字符串"的例子。

2. 用 `outlines` 库做 constrained decoding 是兜底手段，保证 JSON 格式合法，但不能保证语义正确，所以训练数据质量仍然最重要。

3. 生成微调数据后，和 B线确认 DataContext 格式是否和 `loader.py` 实际输出一致（B线可以跑一下 `load_data("data/example_bar.csv")` 打印给你看）。

### 4.3 renderer.py —— 主要工作

#### 替换 render_plot() 主入口

现在的 Mock 直接返回占位图，真实实现要改成：

```python
def render_plot(spec: dict, data_source: str) -> str:
    try:
        # 1. 获取 DataFrame
        df = _resolve_dataframe(data_source, spec)

        # 2. 获取 ThemeConfig（已有 apply_theme 可用）
        theme = apply_theme(
            spec["style_theme"],
            spec.get("style_palette_override"),
        )

        # 3. 分发到子渲染器
        chart_type = spec["chart_type"]
        if chart_type not in RENDERERS:
            raise RenderError(f"不支持的图表类型：{chart_type}")
        fig = RENDERERS[chart_type](df, spec, theme)

        # 4. 保存文件
        out_path = _output_path(chart_type)
        fig.savefig(out_path, dpi=theme.dpi, bbox_inches="tight")
        plt.close(fig)
        return str(out_path)

    except RenderError:
        raise
    except Exception as exc:
        raise RenderError(f"渲染失败：{exc}") from exc
```

`_resolve_dataframe` 和 `_output_path` 辅助函数在文件下方已经写好，直接调用即可。

#### 子渲染器接口规范

每个子渲染器签名统一为：

```python
def _render_xxx(df: pd.DataFrame, spec: dict, theme: ThemeConfig) -> plt.Figure:
```

**建议抽取公共函数 `_apply_theme_to_fig(fig, ax, theme)` 避免重复代码**，负责：字体设置、spines 裁剪、网格、图幅尺寸等。

#### 各子渲染器需要读取的 spec 字段

**bar 图（优先级最高）：**

```
必读：data_x, data_y, style_theme
分组：data_group_by（不为 None 时画分组柱状图）
误差棒：data_error（不为 None 时用该列作为 yerr）
标签：label_title, label_x, label_y
轴范围：axes_y_min, axes_y_max
刻度旋转：axes_x_tick_rotation
参数：params_orientation（"vertical"/"horizontal"）
       params_stacked（True 时堆叠）
       params_sort（"asc"/"desc"/None）
       params_show_values（True 时柱顶标数值）
```

**line 图：**

```
必读：data_x, data_y
多线：data_group_by（不为 None 时按该列分组画多条线）
置信区间：data_error（不为 None 时作为误差带宽度）
标签：label_title, label_x, label_y
轴范围：axes_y_min, axes_y_max
参数：params_markers（True 时画数据点）
       params_smooth（True 时做平滑处理）
```

**heatmap（第二优先级）：**

```
必读：data_x, data_y（数据透视时用）
过滤：data_filter（pandas query 语法，可为 None）
标签：label_title, label_x, label_y
cmap：style_palette_override 为 "coolwarm" 时用该字符串，否则用 theme.palette[0]（或 seaborn 默认）
参数：params_annot（True 时显示单元格数值）
       params_fmt（数值格式字符串，如 ".2f"）
```

**scatter 图：**

```
必读：data_x, data_y
分组：data_group_by
标签：label_title, label_x, label_y
参数：params_alpha（点透明度）
       params_show_regression（True 时叠加回归线）
```

**box 图：**

```
必读：data_x, data_y
标签：label_title, label_x, label_y
参数：params_show_points（"all"/"outliers"/"none"）
       params_notch（True 时缺口箱线图）
```

#### 开发顺序建议

`bar` → `line` → `heatmap` → `scatter` → `box`

前三个是 Demo 必须展示的，后两个按时间决定是否实现。

### 5.4 AgentResponse 结构

C线调用 `agent.process_input()` 后处理返回值，字段含义：

| 字段 | 类型 | 何时有值 | 含义 |
|------|------|----------|------|
| `status` | str | 总是有 | `"ok"` / `"need_input"` / `"error"` |
| `image_path` | str / None | `status == "ok"` | 生成图表的 PNG 路径 |
| `question` | str / None | `status == "need_input"` | 回问用户的文字 |
| `message` | str / None | `status == "error"` | 错误描述 |
| `current_spec` | dict / None | 总是有（调试用） | 当前 PlotSpec 状态 |

**app.py 里处理 AgentResponse 的骨架：**

```python
def on_submit(user_input: str):
    response = agent.process_input(user_input)
    if response.status == "ok":
        return response.image_path, "绘图完成", json.dumps(response.current_spec, indent=2)
    elif response.status == "need_input":
        return None, response.question, json.dumps(response.current_spec or {}, indent=2)
    else:
        return None, f"错误：{response.message}", ""
```

## 6. 三线接口对照表

```
A线产出                          C线消费位置
────────────────────────────────────────────────────────────
generate_spec() -> dict          system/agent.py: process_input()
  首轮：完整 PlotSpec dict         传给 fill_defaults → validate → render_plot
  修改轮：delta dict               传给 merge_delta → fill_defaults → validate


B线产出                          C线消费位置
────────────────────────────────────────────────────────────
load_data() -> (str, str)        system/agent.py: load_data()
  str1: DataContext 字符串         存到 self.data_context，传给 generate_spec
  str2: cache_key                  存到 self.current_cache_key，传给 render_plot

render_plot() -> str             system/agent.py: process_input()
  str: PNG 文件路径                存到 AgentResponse.image_path，UI 展示图片


B线产出                          A线消费位置
────────────────────────────────────────────────────────────
DataContext 格式样例              构造微调训练数据时，instruction 字段用这个格式
  （调用 load_data() 打印查看）
```

### 8.1 新增图表类型

以新增小提琴图（`"violin"`）为例，需要改动 **3 个文件**，validator / agent / UI **一行不动**。

#### 第一步：`schema.py` —— 声明新类型和相关参数

```python
# 1. 添加到 CHART_TYPES
CHART_TYPES = ["bar", "line", "scatter", "box", "heatmap", "violin"]   # ← 追加

# 2. 若新图表有专属参数，添加到 OPTIONAL_DEFAULTS（可复用已有参数则跳过）
OPTIONAL_DEFAULTS = {
    ...
    "params_violin_inner": "box",   # ← 新增：小提琴内部显示方式 "box"/"point"/"none"
}

# 3. 在 CHART_PARAMS 中声明该类型使用哪些 params 字段
CHART_PARAMS = {
    ...
    "violin": ["params_show_points", "params_violin_inner"],   # ← 追加
}
```

**validator.py 无需修改**：它读 `CHART_TYPES` 做校验，新类型自动生效。

#### 第二步：`tools/renderer.py` —— 实现渲染函数并注册

```python
# 1. 实现子渲染器（签名必须与其他子渲染器完全一致）
def _render_violin(df: pd.DataFrame, spec: dict, theme: ThemeConfig) -> plt.Figure:
    import seaborn as sns
    fig, ax = plt.subplots()
    sns.violinplot(
        data=df,
        x=spec["data_x"],
        y=spec["data_y"],
        hue=spec.get("data_group_by"),
        inner=spec.get("params_violin_inner", "box"),   # 读新增的 params 字段
        palette=theme.palette,
        linewidth=theme.line_width,
        ax=ax,
    )
    ax.set_title(spec.get("label_title", ""), fontsize=theme.font_size + 1)
    ax.set_xlabel(spec.get("label_x", ""),    fontsize=theme.font_size)
    ax.set_ylabel(spec.get("label_y", ""),    fontsize=theme.font_size)
    _apply_theme_to_fig(fig, ax, theme)
    return fig

# 2. 注册到 RENDERERS 字典（render_plot 主入口会自动分发，无需修改）
RENDERERS = {
    "bar":     _render_bar,
    "line":    _render_line,
    "scatter": _render_scatter,
    "box":     _render_box,
    "heatmap": _render_heatmap,
    "violin":  _render_violin,   # ← 追加
}
```

#### 第三步：补充测试

在 `tests/test_renderer.py` 里增加：

```python
def test_violin_render_creates_file():
    spec = {
        "chart_type": "violin",
        "data_source": "data/example_bar.csv",
        "data_x": "dataset", "data_y": "accuracy",
        "style_theme": "nature",
        **OPTIONAL_DEFAULTS,   # 从 schema 导入，不要硬编码
    }
    path = render_plot(spec, "data/example_bar.csv")
    assert Path(path).exists()
```

#### 第四步：通知 A线补充训练样本

A线需要在微调数据集里补充至少 20 条 `"chart_type": "violin"` 的样本，否则模型不会生成该类型。

#### 改动文件清单

| 文件 | 改动内容 |
|------|----------|

| `model/generator.py` | A线补充训练样本（函数签名不变） |
| `system/validator.py` | **不需要动**（自动读 `CHART_TYPES`） |
| `system/agent.py` | **不需要动** |
| `ui/app.py` | **不需要动** |

### 8.2 新增预设风格（主题）

以新增 ACL 会议风格（`"acl"`）为例，需要改动 **2 个文件**。

#### 第一步：`schema.py` —— 声明新主题名

```python
STYLE_THEMES = ["nature", "ieee", "neurips", "clean", "morandi", "acl"]   # ← 追加
```

**validator.py 无需修改**：它读 `STYLE_THEMES` 做校验，新主题自动生效。

#### 第二步：`tools/themes.py` —— 添加 ThemeConfig

在 `THEMES` 字典里追加新主题的完整配置：

```python
THEMES: dict[str, ThemeConfig] = {
    ...   # 已有的五个主题
    "acl": ThemeConfig(
        font_family="Times New Roman",
        font_size=10,
        line_width=0.8,
        figure_width=6.0,       # ACL 论文推荐图幅（单栏）
        figure_height=3.75,
        dpi=300,
        spines=["left", "bottom"],
        grid=False,
        grid_style="--",
        legend_frameon=True,
        legend_fontsize=9,
        palette=["#2196F3", "#F44336", "#4CAF50", "#FF9800", "#9C27B0"],
    ),
}
```

`apply_theme("acl")` 随即可用，无需修改其他函数。

PALETTE_OVERRIDES = ["morandi", "nature_d", "tab10", "coolwarm", "pastel"]   # ← 追加

PALETTES: dict[str, list[str] | str] = {
