## 场景全景列表

### 设计原则

每个场景要有**领域真实性**——数据的分布、量级、列名风格都要符合该领域的实际情况，不能用同一套随机参数套所有场景。同时要有**内部变异**——同一场景的不同实例，规模、列名、数值分布都要不一样。

---

### 第一类：NLP / 文本模型评测

**场景1：模型×数据集基准测试（长表）**
4-8个模型，3-6个数据集，指标1-2个（accuracy/F1/BLEU等），含或不含std。
数值分布：不同数据集难度差异大（CoLA类60-70，SST-2类90-96），模型间差距2-8个点。
变异：指标列名（acc/accuracy/score/F1/BLEU/ROUGE-L），模型列名（model/method/system/approach），有时含参数量列（params_M）。

**场景2：消融实验**
基础模型+3-6个变体（去掉某个组件），1-3个指标。
数值分布：完整模型最高，每去掉一个组件下降1-6个点，不同组件影响差异大（关键组件掉5+，次要组件掉1-2）。
变异：variant列名（variant/ablation/setting/configuration），有时含参数量或推理速度。

**场景3：不同训练数据量的学习曲线（横截面）**
5-8个数据量节点（100/500/1k/5k/10k/50k/100k），2-3种方法对比。
数值分布：对数增长趋势，小数据量时方法间差距大，大数据量时收敛接近，有波动。

**场景4：Few-shot性能（k-shot对比）**
k取0/1/2/4/8/16/32，2-4种方法。
数值分布：0-shot到1-shot跳跃大，之后增长放缓，不同方法的零样本能力差异显著。

**场景5：多语言评测**
3-6种语言，1-2个模型，2-3个指标。
数值分布：英语最高（85-95），高资源语言（德/法/中）次之（75-88），低资源语言（斯瓦希里/泰语）差距明显（55-75）。

---

### 第二类：CV / 图像模型

**场景6：图像分类基准（含计算成本）**
5-8个模型，top1/top5 accuracy，params_M，FLOPs_G，推理速度（img/s）。
数值分布：精度-效率权衡明显，轻量模型精度低速度快，大模型反之，ViT类和CNN类各有优势区间。

**场景7：目标检测（mAP多阈值）**
4-6个检测器，mAP@0.5、mAP@0.75、mAP@0.5:0.95，推理时间（ms）。
数值分布：三个阈值数值依次递减（约65/48/42），模型间在严格阈值下差距更大。

**场景8：分割任务（多类别IoU）**
3-5个方法，overall mIoU + 4-8个类别IoU。
数值分布：背景类IoU高（90+），稀有类别IoU低（30-50），不同方法在不同类别上各有强弱。

**场景9：数据增强消融**
基础方法+4-6种增强策略（随机裁剪/颜色抖动/mixup/cutmix等），1-2个指标。
数值分布：单一增强提升2-4%，组合增强提升叠加但有边际递减。

---

### 第三类：训练过程

**场景10：单模型训练曲线**
epoch或step为X轴，train_loss + val_loss（必有），可能含val_acc、lr。
数值分布：loss从2.x指数下降到0.x，前期快后期慢，val_loss在某点后略高于train_loss（过拟合迹象），lr如果有则是阶梯下降或余弦曲线。

**场景11：多模型训练曲线对比**
3-5个模型，同一指标（val_loss或val_acc），对比收敛速度和最终性能。
数值分布：不同模型收敛速度差异明显，最终性能差距1-5%。

**场景12：不同学习率/批大小的训练曲线**
4-6个超参数配置，val_loss随epoch变化。
数值分布：学习率过大→振荡不收敛；过小→收敛极慢；最优配置居中。波动程度随配置不同。

---

### 第四类：超参数搜索

**场景13：双因子网格搜索（热力图场景）**
学习率×批大小，或正则化强度×dropout，5×4或4×4的矩阵。
数值分布：最优区域在中间某个角落，边缘区域性能显著下降，有轻微随机噪声。

**场景14：单因子扫描（折线图）**
1个超参数（dropout/weight_decay/hidden_size/num_layers），4-8个取值，1-2个指标。
数值分布：倒U形（dropout/正则化）或单调递增后平台（模型大小）。

**场景15：随机搜索结果（散点图场景）**
50-200次随机配置，每次记录2-3个超参数值+验证集性能。
数值分布：大多数配置性能中等，少数极好或极差，参数与性能的相关性强弱不一。

---

### 第五类：效率与资源

**场景16：推理速度-精度权衡**
8-15个模型，X轴params_M或FLOPs，Y轴accuracy，散点图场景。
数值分布：整体正相关但非线性，存在帕累托前沿，某些模型效率异常高（创新点）。

**场景17：量化/剪枝实验**
原始模型+4-6个压缩比例（25%/50%/75%压缩），精度vs模型大小。
数值分布：轻度压缩精度几乎不变，重度压缩精度骤降，压缩比例和大小成比例但精度非线性。

**场景18：不同硬件/框架的推理延迟**
3-5种配置（CPU/GPU/TPU或PyTorch/TF/ONNX），batch_size从1到128。
数值分布：GPU在大batch时优势显著，CPU延迟随batch线性增长，GPU在小batch时开销相对更高。

### 第六类：科学实验（非AI）

**场景19：药物/干预实验（RCT风格）**
对照组+2-4个处理组，2-4个时间点（baseline/week4/week8/week12），response_rate + se + n。
数值分布：对照组基本不变，处理组随时间增长但有饱和，不同处理组效果量差异大，se随样本量增大而减小。

**场景20：物理/化学测量**
X轴是物理量（温度/浓度/压力/频率），Y轴是响应变量，多条件对比。
数值分布：明确的物理关系（线性/指数/S形），测量误差小（<5%），不同条件下的响应函数形状相似但参数不同。

**场景21：生物学重复实验**
3-5个处理组，每组3-6次生物学重复，均值+SEM。
数值分布：组间差异是信号，组内变异（SEM）相对较小，但不同处理组的SEM可能不同（方差不齐）。

---

### 第七类：社会科学 / 调查数据

**场景22：问卷Likert量表结果**
5-8个题目，每题5个选项的百分比，或多个群体的平均分。
数值分布：每行百分比加和100%，不同题目的分布形态不同（有的偏正面，有的两极化）。

**场景23：年度趋势数据**
5-10年，3-5个类别，数量或比例随时间变化。
数值分布：有明确趋势（增长/下降/先增后降），不同类别趋势方向可能不同，有随机年际波动。

**场景24：地区/群体对比**
6-12个地区或群体，2-4个指标，可能含排名变量。
数值分布：指标间可能正相关也可能负相关（如GDP高但幸福感不一定高），分布有明显的头部效应。

---

### 第八类：宽表格式（需要特殊处理）

**场景25：模型×数据集宽表**
行是模型（4-6行），列是数据集名称（3-6列数值列），无分组列。
这是情况一的典型形态，对应场景1的宽表版本。

**场景26：方法×指标宽表**
行是方法，列是指标名称（accuracy/F1/BLEU/ROUGE等），每格一个数值。

**场景27：时间×类别宽表**
行是年份/月份，列是类别名称，每格是该时间该类别的数值。

---

**场景28：多次随机种子实验结果**
同一方法用5-10个不同随机种子跑，记录每次的最终性能。
```
method, seed, accuracy, f1
BERT, 42, 93.2, 92.8
BERT, 123, 93.8, 93.1
BERT, 456, 92.9, 92.5
RoBERTa, 42, 94.1, 93.7
...
```
每个方法有5-10行原始数据，box图展示分布和稳定性。

**场景29：k折交叉验证结果**
5-10折，3-5个方法，记录每折的性能。
```
method, fold, accuracy
BERT, fold1, 92.1
BERT, fold2, 93.4
...
```
数值分布：折间波动1-4%，不同方法的稳定性不同（std差异明显）。

**场景30：系统延迟/响应时间分布**
4-6个系统，每个系统50-200次测量，记录每次延迟。
```
system, request_id, latency_ms, batch_size
SystemA, 1, 12.3, 32
SystemA, 2, 11.8, 32
...
```
数值分布：长尾分布（大多数请求快，偶尔有慢请求），不同系统的尾延迟差异大。

**场景31：多数据集上的性能分布**
1-3个方法，在15-30个数据集上各跑一次，box展示跨数据集的性能分布。
```
method, dataset, accuracy
BERT, dataset_1, 91.2
BERT, dataset_2, 78.4
BERT, dataset_3, 85.7
...
```

---

**场景32：模型×数据集性能矩阵**
和场景26类似但专门为heatmap设计——行是方法，列是数据集，值是性能分数，格子里标注数值。适合方法数×数据集数在4×4到8×8之间的矩阵。

**场景33：相关性矩阵**
5-8个指标（accuracy/F1/BLEU/ROUGE等）两两之间的相关系数，值域-1到1，适合coolwarm配色。

**场景34：Scaling Law**
模型参数量（log scale）vs性能，3-5条曲线（不同任务或数据量），每条10-20个点。
数值分布：对数线性关系，各条线斜率相近但截距不同，有轻微噪声。这个场景同时测试log轴的使用。
